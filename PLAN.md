# 台股板塊輪動交易系統 — 定案計畫 (v1)

> 路線：**方案 C（先 MVP 驗證 edge，再分階段加回完整功能）**
> 起手模組：**回測骨架**
> 做多落地：**板塊內市值前三大龍頭股，均分資金**
> 情緒訊號：**行為金融「反應過度 / 反應不足」條件式設計**（非單純買賣）

---

## 0. 核心理論基礎（情緒模組的靈魂）

參考 Fuller & Thaler Asset Management（Thaler = 2017 諾貝爾獎）與行為金融學術研究
（Barberis-Shleifer-Vishny 1998、Hong-Stein 1999、Zurich 2024 news-sentiment 研究）：

市場只有兩種可被預測的投資人犯錯，我們只賺這兩種錢：

- **反應過度 (Overreaction)**：高關注度 + 情緒化 + 擁擠 → 未來**均值回歸** → 做反向。
- **反應不足 (Underreaction)**：低關注度 + 安靜 + 剛點火 → 未來**延續動能** → 做順向。

> 投顧情緒不是「分數越高越買」。投顧在 YouTube 大力吹捧、情緒瘋狂、散戶號召力高
> = 典型「高關注 + 情緒化」= **反應過度特徵 = 偏反轉（空）**。
> 投顧低調剛提到、尚未成為全場焦點 = **反應不足 = 偏動能（多）**。
> 情緒分數必須與「擁擠度 / 關注度」交互，輸出**條件式 (regime-conditional) 訊號**。

這同時消除了「進場追情緒、風控躲情緒」的內部矛盾：進場與風控共用同一套過度/不足邏輯。

## 1. 系統架構與資料流（每週五收盤後）

### 1.1 數據採集（Fri 16:00–18:00）
- **YouTube 字幕流**：目標投顧頻道過去 7 天影片字幕。
  - 嚴格記錄影片「發布時間戳」，回測對齊用，杜絕 look-ahead。
- **盤後籌碼流**：7 大科技板塊（先進封裝、IC設計、散熱、PCB、組裝、記憶體、面板）
  的「融資餘額、融券餘額、當沖成交量」。
- **市場日 K 流**：7 大板塊 + 加權指數 (TAIEX) 日線。

### 1.2 資料庫防禦機制
- 斷線重試（指數退避）。
- **完整性檢查**：任一板塊缺當沖量/收盤價即發警報，**禁止 NaN 進入下一步**。
- 資料落地 **SQLite**，附 schema 與資料字典。

## 2. 特徵工程

### 2.1 LLM 語意特徵（商業 API）
- 板塊過濾：本週完全沒被投顧提過 → 該板塊情緒相關特徵填 0。
- 結構化輸出（JSON schema 強制）：每板塊輸出
  - `bullish` 強力看多程度 0–5
  - `retail_mobilization` 散戶號召力 0–5
  - `emotion_intensity` 情緒瘋狂度 0–5
- **反諷 / 反向情緒辨識**：Prompt 加入規則，要求 LLM 結合上下文判斷老師真實意圖（叫買還是叫賣）。
- **JSON 解析防呆**：解析失敗 → 重試 → 仍失敗則該板塊情緒特徵標記為缺失並警報（不可硬塞 0 假裝有資料）。

### 2.2 計量特徵
- **滾動 OLS 去大盤化**：過去 60 日日資料，跑板塊對 TAIEX 回歸，取**殘差報酬率**。
- **殘差動能（取代二階加速度）**：殘差 20/60 日累積報酬、殘差 Sharpe。
  - 註：原「二階加速度 a」訊噪比過低，MVP 階段移除；若後續驗證有貢獻再加回。
- **當沖擁擠度 Z-score**：當沖週轉率的橫斷面 + 時序 z-score（= 行為框架的「關注度」代理）。
- **融資維持率 / 融資變化**。

### 2.3 情緒條件式合成（核心）
依「情緒分數 × 擁擠度/關注度」分流：
- 情緒中高 + 擁擠度低 → **反應不足 → 動能多單訊號**。
- 情緒極高 + 擁擠度極高 → **反應過度 → 避開 / 反向**。
- 情緒極低 + 擁擠度高 → **反應過度 → 逢低回歸多單**。

## 3. 排序與輸出

### 3.1 模型（分階段，避免過擬合）
- **階段 1（MVP）**：**透明加權打分**（z-score 加總），不用 ML。
  - 理由：只有 7 板塊 × 每週 1 樣本，樣本數 ≈ 特徵維度，ML 必過擬合。
  - 先確認「訊號本身有沒有 edge」。
- **階段 2**：edge 確認後，逐一加入特徵，每加一個就驗證正貢獻。
- **階段 3**：樣本/特徵成熟後才換 **LightGBM Rank**。

### 3.2 輸出
- 下週 7 大板塊預測排名（1–7）。

## 4. 交易執行與資金管理

### 4.1 進場
- 只做多排名 **Top 1 與 Top 2** 板塊。
- 落地：各買該板塊**市值前三大龍頭股，均分資金**（分散個股非系統性風險）。
- 時間點：週一開盤輪動。
- **滑價防禦**：用「開盤後 15 分鐘 VWAP」或分批限價單，不追當天最高點。

### 4.2 兩道保命鈕（三方案皆保留）
- **第一道｜擁擠度踩踏停損（週中即時）**：持有板塊當沖週轉率突破 65%
  且股價跌破「過去 20 日散戶融資進場均價 ×130%（融資斷頭警戒線）」
  → 盤中強制清倉該板塊 50%~100%。
- **第二道｜總體最大回撤（Max Drawdown Stop）**：單週帳戶整體虧損達 **8%**
  → **降至半倉、觀察一週**（原「斷電一週」改為降槓桿，避免錯過崩盤後反彈）。

## 5. 落地三階段時程

### 階段一：歷史回測與統計防禦（第 1–2 週）—— 現在開始
- 拿 2024–2026 台股歷史資料跑完整回測。
- **必須扣除**：證交稅（當沖 0.15% / 一般 0.3%）+ 券商手續費 + 滑價。
- **A/B 必跑**：情緒「順向 vs 反向 vs 條件式」三版本對照，讓資料決定方向。
- 通過門檻：淨報酬戰勝大盤 **且** Sharpe > 1.5。未過 → 不上線。
- ⚠️ 注意：用今日 LLM 讀舊字幕有隱性 look-ahead，回測 Sharpe 會虛高。
  **回測結論當參考、不當聖旨**；真正可信的績效來自階段二。

### 階段二：紙上模擬（第 3–6 週，延長至 8 週）
- 模型對接 SQLite，每週五跑真實訊號，週一記錄模擬買價（不投真錢）。
- 目標：解決所有工程 bug（漏抓字幕、JSON 死當、管線斷裂）。
- 4 個樣本證明不了任何事 → **至少 8 週**。

### 階段三：小資金實戰（第 9 週起）
- 投入「全賠光也不影響生活」的金額。
- 嚴格執行週頻輪動，克服心理偏誤。
- 唯一考核指標：**對帳單 P&L**。

## 6. 立即動工：回測骨架（Step 1）

最小可跑骨架，先驗證 edge，元件如下：
1. 資料載入層（先用歷史日 K + 板塊籌碼；字幕情緒先用 placeholder / 後補）。
2. 滾動 OLS 去大盤化 → 殘差報酬。
3. 殘差動能特徵 + 擁擠度 z-score。
4. 加權打分排序（Top 1/2 選股）。
5. 交易模擬：稅費 + 滑價 + 週頻輪動 + 龍頭股均分。
6. 績效：淨報酬、Sharpe、最大回撤、vs TAIEX。
7. A/B 開關：情緒順向 / 反向 / 條件式。

> 待確認：歷史資料來源（是否已有 CSV / API token / 既有 DB？）。
